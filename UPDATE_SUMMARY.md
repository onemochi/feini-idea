# 菲尼合同引擎 OCR 功能升级总结

## 📋 更新概述

本次更新为菲尼合同引擎的 PDF 识别功能添加了两个重要的用户控制开关，大幅提升了用户体验和灵活性。

---

## ✨ 新增功能

### 1. 🪟 Windows OCR 开关

**功能说明**：在 EXE 版本中控制是否使用 Windows 系统 OCR 进行识别

**特性**：
- ✅ 默认关闭，使用 tesseract.js（保证兼容性）
- ✅ 用户可手动启用，切换到 Windows 系统 OCR
- ✅ 状态持久化，下次打开自动恢复
- ✅ 实时反馈，切换时显示日志和更新提示

**优势**：
- **速度更快**：Windows OCR 识别速度是 tesseract.js 的 5-10 倍
- **离线可用**：不需要下载模型，即开即用
- **中文友好**：对中文扫描件识别更准确
- **用户可控**：想用哪个引擎用户自己决定

### 2. ⚡ OCR 模型预加载开关

**功能说明**：控制是否在页面加载时预先下载 tesseract.js 模型

**特性**：
- ✅ 默认关闭，避免自动消耗流量
- ✅ 启用后后台自动预加载（不阻塞其他操作）
- ✅ 预加载完成后首次识别无需等待
- ✅ 智能跳过：启用 Windows OCR 时自动跳过预加载

**优势**：
- **即开即用**：首次识别无需等待下载模型（节省 30-120 秒）
- **后台加载**：不影响其他操作，用户无感知
- **智能优化**：自动判断是否需要预加载，不浪费资源
- **用户可控**：网速慢或流量有限时可关闭

## 🎯 使用场景推荐

### 场景 1：Windows 10/11 用户（EXE 版本）
```
✅ Windows OCR: 开启
❌ 预加载模型: 关闭
```
**原因**：Windows OCR 更快更准，不需要 tesseract.js

### 场景 2：浏览器用户 + 网速快
```
❌ Windows OCR: 关闭（不可用）
✅ 预加载模型: 开启
```
**原因**：预加载后首次识别无需等待

### 场景 3：浏览器用户 + 网速慢/流量有限
```
❌ Windows OCR: 关闭（不可用）
❌ 预加载模型: 关闭
```
**原因**：避免自动消耗流量，按需加载

### 场景 4：对比测试两种引擎
```
根据需要切换 Windows OCR 开关
✅ 预加载模型: 开启（保证 tesseract.js 随时可用）
```
**原因**：两种引擎都可用，可随时切换对比效果

## 🔧 技术实现

### 代码改动
- **修改文件**：`index.html`（单文件修改）
- **新增代码**：约 150 行
- **影响模块**：PDF 识别拆分功能
- **向后兼容**：✅ 是（默认关闭，不影响现有行为）

### 核心函数
1. **Windows OCR 控制**：
   - `saveWinOcrToggle()` - 保存开关状态
   - `initWinOcrToggle()` - 初始化开关
   - `isWindowsOcrEnabled()` - 检查是否启用
   - 修改 `initOcrAdapter()` - 根据开关选择引擎
   - 修改 `detectOcrEngine()` - 更新状态提示

2. **模型预加载**：
   - `savePreloadOcrToggle()` - 保存开关状态
   - `initPreloadOcrToggle()` - 初始化开关并触发预加载
   - `isPreloadEnabled()` - 检查是否启用
   - `startPreloadOcrModel()` - 执行预加载
   - 修改 `initOcrAdapter()` - 优先使用预加载的 workers

### 数据持久化

## ⚠️ 注意事项

### Windows OCR
1. **仅 Windows 10/11 可用**：其他系统自动降级到 tesseract.js
2. **需要语言包**：未安装中文语言包时可能识别效果不佳
3. **默认关闭**：为了兼容性，需要用户手动开启

### 预加载模型
1. **消耗流量**：首次下载约 30MB 模型文件
2. **占用内存**：预加载会提前占用 30-50MB 内存
3. **默认关闭**：避免自动消耗用户流量
4. **智能跳过**：启用 Windows OCR 时自动跳过预加载

## 🐛 问题排查

### Q: 开启 Windows OCR 后还是很慢？
**A**: 检查以下几点：
1. 确认是 Windows 10/11 系统（Win7/8 不支持）
2. 检查是否安装了中文语言包
3. 查看终端日志，确认使用的是 Windows OCR 而不是 tesseract.js
4. 如果提示"不可用"，说明系统缺少必要组件

### Q: 开启预加载后没有效果？
**A**: 检查以下几点：
1. 查看终端日志，确认预加载是否完成
2. 确认网络连接正常（预加载需要下载模型）
3. 如果同时启用了 Windows OCR，预加载会自动跳过
4. 刷新页面，重新尝试

### Q: 预加载很慢怎么办？
**A**: 
1. 预加载速度取决于网速（30MB 文件）
2. 可以关闭开关，改为按需加载
3. 或者使用 Windows OCR（EXE 环境）
